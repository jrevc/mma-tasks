File & Folder Naming Conventions

Since websites are viewed over the internet, web developers upload files to a web server.

Some web servers are finicky about lowercase and uppercase letters. Some don't know how to handle spaces in filenames. Some strangely crash when it tries to read non alpha-numeric in filenames.

Because of this, web developers have thought to use naming conventions for files and folder as follows:

All lowercase letters

  • It is best to use all lowercase letters so that you will never have to wrestle with case-sensitive web servers. By using all lowercase, you can be sure all filenames and folders are consistent.
  • One of the thinks overlooked is the extension, the thing that follows the dot. (e.g .jpg, .png. .html) Those, too, should be in lowercase.

No spaces!

  • There should absolutely be no spaces in the filename. When a web server encounters a space, sometimes it thinks the name is done. So, to avoid this, we never put spaces in any of the filenames or folders.

Only Alphanumeric characters, dash, and underscore are used.

  • Use only alphabets and numerals. The only other two characters that can be used are the dash, and the underscore.
  • abcdefghijklmnopqrstuvwxyz
  • 0123456789 _ -
